## Independent Review Follow-Up

A fresh independent review pass after the first implementation found additional correctness and maintainability gaps. These were verified and corrected before final verification:

- SCCP join handling now marks entry/runtime values and unknown edge arguments overdefined.
- Strict release classification no longer mistakes echoed `allow-missing` command text for a skipped phase.
- Tagged enum payload layout now uses per-case union payloads, supports source-typed stdlib payloads, and emits flat field facts required by Proof-MIR compatibility checks.
- Proof-MIR enum construct lowering no longer invents payload metadata for dynamic enum tags.
- HIR enum constructor lowering has a reachable fail-closed path for missing payload field records.
- HIR enum-constructor fail-closed paths now register returned error expressions in the body index, and the enum-constructor mismatch diagnostic is part of the stable HIR diagnostic catalog.
- OptIR pipeline change detection caches stable fingerprints and fast-paths unchanged references.
- Enum layout code was decomposed into payload and diagnostic helper modules to satisfy runtime line-count caps.
- UEFI AArch64 `.pdata` unwind entries now use image-relative `addr32nb` relocations, and PE parsing validates 8-byte AArch64 exception entries instead of accepting range-only exception directories.
- OptIR enum payload-load verification now uses a forward must-analysis for transitive tag-case dominance instead of checking only immediate predecessor switch edges.
- OptIR memory SSA now traverses CFG-reachable blocks and records deterministic unknown-merge states when only some join predecessors define a range.
- OptIR vectorization now reserves operation/value IDs through canonical fresh allocation, and `operationMap` rejects duplicate operation IDs instead of silently collapsing them.
- Production `as never` sentinel casts were removed across `src`; attempt lowering carries real expression IDs into canonicalization, and the WCR-19 audit now scans all runtime source.
- The pass-manager architecture audit now verifies that `runOptIrPassPipeline` is the production scheduling path and that `OptIrPassContext` is constructed only in `pass-manager.ts`.
- UEFI AArch64 unwind metadata now derives the framed entry thunk length from source symbols, decodes to the real 64-byte thunk body, omits non-synthetic `frameless-leaf` source functions from linked unwind metadata, and carries backend frame-size/saved-register facts for non-leaf `serializable-unwind` source functions.
- The AArch64 object unwind-record model was extracted to `src/target/aarch64/backend/object/object-unwind-record.ts` so the added typed metadata does not grow the grandfathered `object-module.ts` giant file.
- LICM no longer accepts a public `loopOperationIds` filter; it derives loop operations from the loop tree and retains dependency-ordered hoisting.
- AArch64 exception-directory parsing now rejects all-zero entries inside non-empty `.pdata` directories instead of treating them as ignorable padding.
- OptIR construction now rejects duplicate operation IDs before building the verifier input table, preventing last-write-wins collapse at the construction boundary.
- Import-cycle diagnostics are frontend-blocking errors, and the typed frontend stage rejects real cyclic module graphs before semantic execution.
- Release phase skip classification now uses explicit line-level skip markers and known skip stable-details, not arbitrary prose containing "skipped".
- The UEFI package frontend now uses the parser-backed module loader and rejects direct and multi-module package import cycles at the target boundary.
- HIR/proof-check private predicate identity now includes call arguments, private place, and generation; semantic private-transition surfaces emit one entry per private input.
- Owner-sensitive HIR metadata lowerers now fail closed through `HIR_MISSING_OWNER_FUNCTION`, and program-level HIR contexts use an explicit `program` owner instead of function-zero sentinels.
- Reproducibility manifests now include source input digests and per-build validation report digests, and isolated build-pass comparison checks those hashes.
- OptIR pass results carry concrete `OptIrProgram` values, and fixpoint convergence honors explicit pass `changed` flags.
- Production stack promotion now passes complete structural escape-analysis evidence, promoting only non-escaping activation stack locals and preserving ordered-effect/external escape boundaries.
- SCCP now keeps foldable operations and edge arguments unknown until source values become constant or overdefined, so constants are not lost merely because producers have higher operation IDs.
- `verify:stdlib` now parses the documented public module list from `docs/stdlib/compatibility.md`, and compatibility tests consume that verifier source of truth instead of duplicating the list.
- The stack-promotion escape evidence policy was extracted to `src/opt-ir/passes/stack-promotion-escape.ts` after the subsystem maintainability audit caught `pipeline-steps.ts` exceeding the OptIR line cap.
- The OptIR pass manager now runs structural verification for schedule entries whose contract requires after-run verification, records `after-pass` checkpoints, and fails closed on invalid pass output.
- `verify:stdlib` now includes a public-surface case that checks documented exports and enum cases against stdlib source declarations before compile smoke cases run.
- Proof-MIR canonicalization now rejects missing monomorphized function-instance metadata instead of synthesizing function-zero/item-zero placeholders.
- Iterator Proof-MIR integration fixtures now construct complete monomorphized function instances with stable source IDs, so snapshot coverage no longer depends on the removed zero-ID fallback.
